# fire-tools 설계 문서

## 개요

FIRE(Financial Independence, Retire Early) 관련 재테크 웹 도구 모음 사이트. 하나의 Next.js 앱에 허브 페이지 + 계산기 3종을 포함한다. 기존 FIRE 은퇴 계산기(Streamlit Cloud)는 외부 링크로 연결.

## 기술 스택

- **프레임워크:** Next.js 14+ (App Router)
- **스타일링:** Tailwind CSS + shadcn/ui
- **차트:** Recharts (shadcn/ui 차트 컴포넌트)
- **배포:** Vercel
- **언어:** TypeScript

## 페이지 설계

### 1. 허브 페이지 (`/`)

본인용 내비게이션. 도구 카드 목록으로 구성.

- **FIRE 은퇴 계산기** → Streamlit Cloud 외부 링크
- **연봉 실수령액 계산기** → `/salary`
- **전월세 전환 계산기** → `/rent-convert`
- **대출 상환 계산기** → `/loan`

카드에는 도구 이름 + 한 줄 설명 + 링크. 도구 추가 시 카드만 추가.

### 2. 연봉 실수령액 계산기 (`/salary`)

**기준:** 2026년 과세표준

**입력:**
- 연봉 (만원)
- 비과세액 (월, 기본값 20만원 — 식대)
- 부양가족 수 (본인 포함, 기본값 1)
- 20세 이하 자녀 수 (기본값 0)

**계산 항목:**
- 국민연금 (4.5%, 상한 월 590만원 기준)
- 건강보험 (3.545%)
- 장기요양보험 (건강보험의 12.95%)
- 고용보험 (0.9%)
- 소득세 (간이세액표 기반 근사 계산)
- 지방소득세 (소득세의 10%)

**출력:**
- 월 실수령액 (큰 숫자로 강조)
- 공제 항목별 금액 표
- 연봉 대비 실수령 비율
- 기준 연도 배지 표시
- "근사치이며 실제와 차이가 있을 수 있습니다" 안내
- **연봉별 실수령액 참고표** (페이지 한쪽에 상시 표시)
  - 5,000만원 ~ 1억 5,000만원 구간, 500만원 단위
  - 부양가족 1인(본인) 기준 월 실수령액
  - SSR로 렌더링하여 검색 엔진에 노출 (SEO 핵심 콘텐츠)

### 3. 전월세 전환 계산기 (`/rent-convert`)

**입력:**
- 전환 방향 선택: 전세→월세 / 월세→전세
- 보증금 (만원)
- 월세 (만원) — 월세→전세 시
- 전세금 (만원) — 전세→월세 시
- 전월세 전환율 (%, 기본값: 한국은행 기준)
- 예상 투자 수익률 (%, 기본값 4%) — 기회비용 계산용

**계산:**
- `월세 = (전세금 - 보증금) x 전환율 / 12`
- 또는 역으로 전세금 산출
- 기회비용 비교: 여유 자금 투자 수익 vs 월세 지출

**출력:**
- 변환 결과 (월세 또는 전세금)
- 연간 월세 총액
- **손익 분석**
  - 전세→월세: 여유 자금 투자 수익(연) vs 연간 월세 → 연간 이득/손해 금액
  - 월세→전세: 절약되는 연간 월세 vs 추가 보증금 기회비용 → 연간 이득/손해 금액
  - "전세가 연 ○○만원 이득" 또는 "월세가 연 ○○만원 이득" 형태로 표시
- 기준 연도 + 전환율 출처 안내

### 4. 대출 상환 계산기 (`/loan`)

**입력:**
- 대출 원금 (만원)
- 연 이자율 (%)
- 대출 기간 (년)
- 거치 기간 (개월, 기본값 0)

**3가지 상환 방식 동시 비교:**

| 방식 | 설명 |
|------|------|
| 원리금균등 | 매달 같은 금액. 가장 일반적 |
| 원금균등 | 매달 원금 동일 + 이자 감소. 총이자 적음 |
| 체증식 | 매달 상환액 증가. 초반 부담 적음 |

**출력:**
- 3가지 방식 요약 비교 카드 (월 초회/최종 상환액, 총이자, 총상환액)
- 비교 차트: 월별 상환액 추이 (3개 라인), 원금/이자 비율 변화
- 총이자 차이 안내 (예: "원금균등이 원리금균등보다 총이자 320만원 적음")
- 각 방식 유리한 상황 가이드
- 상세 상환 스케줄 표 (접어두기)
- **조기상환 손익 분석**
  - 추가 입력: 조기상환 시점 (몇 개월차), 조기상환수수료율 (%, 보통 1~1.5%)
  - 계산: 조기상환수수료 vs 남은 기간 이자 총액
  - "지금 갚으면 ○○만원 이득" 또는 "수수료가 더 크므로 ○○만원 손해" 형태로 표시
  - 손익분기점 표시: "○○개월차 이후 갚으면 이득" (수수료 < 남은 이자가 되는 시점)
- 기준 연도 안내

## SEO 전략

- 각 페이지별 Next.js `metadata` (title, description, keywords — 한국어 검색 키워드)
- OG 태그 (title, description, image)
- `robots.txt`, `sitemap.xml` 자동 생성
- 시맨틱 HTML (`<main>`, `<section>`, `<h1>` 등)

## 애드센스 준비

- `layout.tsx`에 애드센스 스크립트 자리 (주석 처리)
- 각 계산기 페이지에 광고 슬롯 2곳 예약 (결과 영역 위, 페이지 하단)
- 실제 연동은 나중에 주석만 해제

## 기준 연도 관리

- 각 계산기 상단에 배지로 "2026년 기준" 표시
- `lib/constants.ts`에 연도별 요율/세율 모아두어 매년 업데이트 용이하게

## 계산 방식

- 모든 계산은 클라이언트 사이드 (서버 API 불필요, 순수 함수)
- 계산 로직은 `lib/`에 분리하여 테스트 가능하게
- 입력 → 즉시 결과 반영 (리액티브)
